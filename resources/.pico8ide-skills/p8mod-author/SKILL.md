# p8mod Author

Create valid `.p8mod` files for Pico-8 IDE. A `.p8mod` is a plain-text PICO-8 cartridge with optional `__meta__` and `__i18n__` sections after the standard cartridge sections. Prefer `.p8mod` when the user wants Pico-8 IDE editing, localized games, translation tables, or reusable source cartridges.

## Authoring Goal

Help agents produce more i18n-ready PICO-8 games. When creating a new game, include a small playable Lua loop, useful metadata, and localized visible strings unless the user explicitly wants a non-i18n cartridge.

Prefer creating a new cartridge through Pico-8 IDE's `PICO-8: New Mod Cartridge` command or by modifying an existing valid `.p8mod`. Hand-writing full cartridge files is allowed only when needed, and must be validated before completion.

## Required Structure

```text
pico-8 cartridge // http://www.pico-8.com
version 42
__lua__
<lua source>
__gfx__
<sprite sheet hex>
__gff__
<sprite flag hex>
__map__
<map hex>
__sfx__
<sfx hex>
__music__
<music hex>
__meta__
<optional JSON object>
__i18n__
<optional JSON object>
```

## Rules

- The first line must be exactly `pico-8 cartridge // http://www.pico-8.com`.
- The second line must be `version 42`.
- Standard sections must appear in this order: `__lua__`, `__gfx__`, `__gff__`, `__map__`, `__sfx__`, `__music__`.
- Custom sections, when present, must appear after the standard sections.
- Include all six standard sections even when their bodies are empty or zero-filled.
- End the file with a newline.

## Metadata

Use `__meta__` for Pico-8 IDE metadata:

```json
{
  "title": "My Game",
  "author": "Author Name",
  "template": "default"
}
```

Omit `__meta__` only when title and author are empty and the template is `default`.

## i18n

Use `__i18n__` when Lua calls `tx("key", x, y, color)` or `_txi()`:

```json
{
  "locales": ["en", "zh"],
  "entries": [
    { "key": "title", "translations": { "en": "My Game", "zh": "我的游戏" } }
  ],
  "outputLocale": "en"
}
```

Every entry must provide every locale listed in `locales`. Every `tx("key")` call in Lua must have a matching entry.

When authoring a new i18n game:

- Call `_txi()` from `_init()`.
- Use `tx("key", x, y, color)` for title, menu, status, win, lose, and tutorial text.
- Keep translation keys stable and lowercase, such as `title`, `start`, `score`, and `game_over`.
- Include at least English plus the user's requested locale when no locale set already exists.
- Do not localize code identifiers, only player-visible strings.

## Blank Defaults

- Blank `__gfx__`: 128 lines of 128 `0` hex characters.
- Blank `__gff__`: 2 lines of 128 `0` hex characters.
- Blank `__map__`, `__sfx__`, and `__music__`: section header only is accepted by Pico-8 IDE.
- If using music or SFX, prefer valid PICO-8 text export lines copied from an existing cartridge or generated by a known project helper.

## Verification

- Run the bundled validator after creating or editing a `.p8mod` file:

```bash
node .codex/skills/p8mod-author/scripts/validate-p8mod.mjs path/to/game.p8mod
```

- If Node.js is unavailable, use the Python fallback:

```bash
python3 .codex/skills/p8mod-author/scripts/validate-p8mod.py path/to/game.p8mod
```

- If `python3` is unavailable, try `python` with the same script.
- Both validators use only standard libraries and require no package installation.
- If neither Node.js nor Python is available, inspect the section order, JSON validity, i18n key coverage, and required header lines manually, then report that validator execution was blocked by the environment.
- Treat validator errors as blockers. Fix the file and rerun validation before finishing.

## Runtime Notes

- `.p8mod` stores i18n data as metadata. Pico-8 IDE can generate the Lua i18n runtime during run/export.
- Do not assume a raw `.p8mod` with `tx()` and `_txi()` will run directly in vanilla PICO-8 without Pico-8 IDE export/codegen.
